n

发表于 2016-08-03   |  

n 是 Node.js 的版本管理器，由 Express 框架的作者tj编写的。虽然目前在 github 上只有 4000+ 的 stars ，与其类似的 nvm 相差深远。但我还是比较喜欢 n ，要问我为什么，请听下文分解^^

安装

n 有3种安装方式

  1. npm install -g n

    如果已经安装来 node 可以用这种安装方式。

  2. make install

    编译安装。
    先用git clone 项目。 这里我将不使用默认的安装目录 /usr/local。主要是后面使用 node 和 npm 时要加上 sudo 相当不方便，还有一些其他问题。

  3. `curl -L http://git.io/n-install | bash

用法

n -h 查看具体用法。

这里简单介绍一下常用的用法：
n <version> 就可以安装对应版本号的 nodejs 了。如果已经该版本已经安装那就切换这个版本为当前 nodejs 版本。
直接运行 n 会列出已经安装的版本，用上下键选取需要的版本。
n latest n stable n lts 分别安装 最后一个发布版本、最后一个稳定版本以最后一个长支持版本。
n rm <version> 删除一个版本
n io <version> 安装 io.js 的版本。

安装完 nodejs 或用 n 切换过版本后，不要忘记运行 node -v 验证。
